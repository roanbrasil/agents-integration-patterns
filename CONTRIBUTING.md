# Contributing

This catalog aims to be a rigorous, citable pattern *language* — not just a list. Contributions are welcome, but every pattern must meet the bar below so the catalog stays trustworthy.

## Adding or editing a pattern

Every pattern file in `patterns/**/*.md` must include all required template sections (enforced by CI via `scripts/validate_catalog.py`):

```
## Intent
## Context
## Problem
## Forces                    ← reference the shared F1–F11 vocabulary in FORCES.md
## Solution
## Consequences
## Failure Modes Mitigated   ← map to MAST modes in FAILURE-MAP.md
## Known Uses
## Related Patterns          ← typed links: uses / used-by / alternative-to / refines / conflicts-with
## References                ← real citations only; never fabricate
```

Alexander's bar applies: **no pattern without at least one real-world known use.** Speculative ideas live in a `DRAFTS.md`, not the catalog, and are marked `☆ Proposed` until known uses solidify.

## Every new pattern ships complete

A pattern is not done until it has all three:

1. **Doc** — the expanded template above.
2. **Sample** — a runnable, tested implementation under `samples/`, passing in CI with no API key (use the deterministic fakes in `samples/shared/`). Keep it under ~120 lines.
3. **Diagram** — an SVG in `img/`, generated by `scripts/render_diagrams.py` in the shared visual grammar (see `img/_legend.svg`).

No orphan patterns. A doc without code and a diagram does not merge.

## Running the gates locally

```bash
pip install -e ".[dev,diagrams]"
pytest samples/ -q
python scripts/render_diagrams.py
python scripts/validate_catalog.py
ruff check samples/ scripts/
mypy samples/ scripts/
```

## Maturity ratings

- **★★ Established** — proven across multiple production systems / frameworks.
- **★ Emerging** — real known uses, still stabilizing.
- **☆ Proposed** — promising, limited known uses; use with care.
